# Screenshot Guide

Add your app screenshots here for the GitHub README.

## Required Screenshots

1. **home.png** - Home screen with language selection
2. **translator.png** - Translation screen in action
3. **phrasebook.png** - Phrasebook tab overview
4. **greetings.png** - Festival greetings section (NEW!)
5. **food.png** - Local food discovery cards
6. **emergency.png** - Emergency numbers screen

## How to Take Screenshots

### On Android Device:
1. Run the app: `npm run android`
2. Navigate to each screen
3. Take screenshot: Press **Power + Volume Down** simultaneously
4. Find screenshots in device's Pictures/Screenshots folder
5. Transfer to computer via USB or cloud

### On Android Emulator:
1. Run emulator: `npm run android`
2. Navigate to each screen
3. Take screenshot: Click the camera icon 📷 in emulator toolbar
4. Screenshots saved to desktop automatically

### On iOS Device:
1. Run the app: `npm run ios`
2. Navigate to each screen
3. Take screenshot: Press **Side Button + Volume Up**
4. Find in Photos app → Screenshots album

### On iOS Simulator:
1. Run simulator: `npm run ios`
2. Navigate to each screen
3. Take screenshot: **Cmd + S**
4. Screenshots saved to desktop

## Image Requirements

- **Format**: PNG (recommended) or JPG
- **Dimensions**: Portrait orientation (1080 x 2340 or similar)
- **Size**: < 500KB per image for faster GitHub loading
- **Quality**: Clear, readable text

## Naming Convention

Use exactly these filenames:
- `home.png`
- `translator.png`
- `phrasebook.png`
- `greetings.png`
- `food.png`
- `emergency.png`

## Tips for Great Screenshots

✅ **DO:**
- Use light mode for better visibility
- Show real translations (not placeholders)
- Capture key features in action
- Use a clean, uncluttered screen state

❌ **DON'T:**
- Include personal information
- Show error states
- Use blurry or low-quality images
- Mix light and dark mode screenshots
